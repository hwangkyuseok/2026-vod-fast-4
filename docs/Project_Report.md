# VOD FAST Ad Pipeline - Project Report

## 1. 개요 (Overview)

본 프로젝트는 데이터베이스 대신 JSON 포맷을 데이터 보관소로 사용하여, VOD 콘텐츠 분석(객체 인식, 분위기, 모션 등)을 수행하고 사용자 선호도(나이, 장르 등)와 매칭하여 "가장 적합한 제품 광고"를 "가장 적절한 시간"에 노출하는 FAST (Free Ad-supported Streaming TV) 광고 오버레이 파이프라인입니다.

## 2. 사용 기술 (Technologies)

- **Backend Framework**: `FastAPI` (Python)
- **Computer Vision / AI**:
  - `YOLOv8L` (ultralytics) - 영상 내 객체를 감지 (예: 자동차, 사람, 컵 등) — MVP 15 클래스 기준
  - `OpenCV` (cv2) - 영상 프레임 추출, 프레임간 픽셀 차이 기반 모션 감지 (저모션 구간 탐지)
  - `Silero VAD` (PyTorch) - 딥러닝 기반 음성 활동 탐지 (침묵 구간 탐지)
  - `Whisper` (OpenAI) - 음성 인식 및 소리 탐지 (예정)
  - `Qwen2-VL` / Google Vision - 장면 감정 분석 (상황/감정/욕구 추출)
- **Storage**: JSON File System (`users.json`, `ad_catalog.json`, `vod_metadata.json`), PostgreSQL (`hv02`)
- **Frontend**: `HTML5`, `CSS`, `Vanilla JS` (의존성 없는 순수 프론트엔드 컴포넌트)

## 3. 누적 및 활용 데이터 (Data Points)

### 3.1 사용자 데이터 (`users.json`)

- `age`: 나이 분포 (타겟 연령층 매칭용)
- `favorite_genres` / `favorite_actors`: 선호 장르/배우 (광고 컨텍스트와 매칭)
- `viewing_history`: 사용자 시청 이력 정보 (추가 가중치 계산용)

### 3.2 광고 카탈로그 (`ad_catalog.json`)

- `target_age_min` / `target_age_max`: 광고 타겟 연령
- `target_keywords`: 제품 관련 키워드

### 3.3 비디오 메타데이터 (`vod_metadata.json`)

`video_analyzer.py`를 통해 다음 데이터가 추출/누적됩니다.

- `analyzed_objects`: YOLOv8을 통해 영상 전체에서 발견된 고유 객체 목록
- `dominant_mood`: OpenCV로 프레임 밝기를 측정하여 추정한 다수 분위기 (bright, neutral, dark_suspense)
- `motion_timeline`: 1초 단위로 프레임 변화량을 계산한 모션 점수 맵.점수가 낮을수록(정적일수록) 광고 오버레이를 띄우기 좋습니다.

## 4. 백엔드 결정 로직 (Scoring & Timing)

### 4.1 WHAT : 어떤 제품을 광고할 것인가 (`scorer.py`)

1. **User Score**: 로그인된 사용자의 나이가 광고 제품의 타겟 연령에 포함되면 가점.
2. **Content Score**: 영상 속에서 YOLO로 감지된 객체 목록과, 제품 광고의 `target_keywords`를 교집합 하여 일치하는 항목마다 점수를 부여.
3. **Scene Score**: 장면 분석 모델(Qwen2-VL)이 추출한 상황/감정/욕구와 광고 컨텍스트를 매칭하여 가점.
   => 이 스코어를 합산(`Total Score`)하여 가장 높은 점수의 광고를 선택.

### 4.2 WHEN : 어느 타이밍에 광고할 것인가 (`timing_calculator.py`)

멀티 모델 스코어링 방식으로 최적 광고 타이밍을 결정한다.

| 모델 | 역할 | 점수 기준 |
| ---- | ---- | --------- |
| **OpenCV** | 모션 탐지 | 움직임이 적은 구간 = 높은 점수 |
| **Silero VAD** | 침묵 탐지 | 대사가 없는 구간 = 높은 점수 |
| **Whisper** | 소리 탐지 | 대화 전환점/무음 구간 = 높은 점수 |
| **Qwen/Google** | 장면 분석 | 씬 전환 지점 = 높은 점수 |

각 모델의 점수를 가중 합산하여, 시청자 몰입을 가장 덜 방해하는 타이밍을 선정한다.

## 5. 실행 절차

1. `pip install -r requirements.txt` 도구 설치
2. `python backend/video_analyzer.py` 스크립트를 최초 1회 실행하여 `SampleVideo.mp4`를 기반으로 `vod_metadata.json`을 채웁니다.
3. `python backend/main.py` (또는 `uvicorn backend.main:app`)로 서버를 띄웁니다.
4. 웹브라우저에서 `localhost:8000`으로 접속하여 결과를 확인합니다.

## 6. YOLO 모델 최적화 실험 계획

영상 속 사물 검출 정확도 향상을 위한 YOLO 모델 파라미터 튜닝 및 실험 계획입니다. 파라미터의 경우의 수가 많으므로 우선순위에 따라 단계별로 접근합니다.

### 6.1 1차 우선 실험 파라미터

- **해상도 및 훈련 기본 설정**: `imgsz`, `epochs`, `batch`, `lr0`, `optimizer`
- **데이터 증강 (Augmentation)**: `mosaic`, `scale`, `hsv_h`, `hsv_s`, `hsv_v`
- **모델 구조 및 학습 방법**: `model size` (n/s/m), `pretrained / fine-tuning` 여부
- **임계값 조정 및 추가 튜닝**: `conf`, `iou`, `class` 수 조정, `anchor auto tuning`

### 6.2 헬로비전 프로젝트 특화 파라미터 (영상 속 사물 검출)

- **아주 중요한 파라미터**: `imgsz`, `frame_interval`, `conf`, `iou`, `epochs`, `lr0`, `mosaic`, `scale`, `model size`, `class definition`, `fine-tuning dataset quality`
- **상황별 특이 파라미터**:
  - **작은 객체가 많을 때**: `imgsz` 상향, `anchor` 튜닝, `mosaic` 사용
  - **TV 장면 등이 어두울 경우**: `hsv_v`, 밝기 계열 augmentation
  - **객체가 화면 구석이나 멀리 있을 경우**: `scale` 조정, 고해상도 입력 모델 사용

### 6.3 추천 실험 순서 (Step-by-Step)

- **Step 1. Baseline 만들기**: 기본 모델 1개로 베이스라인 성능 측정
- **Step 2. 입력 크기 실험**: `imgsz` (320, 640, 960 등)
- **Step 3. 학습 설정 실험**: `epoch`, `batch`, `lr0`, `optimizer`
- **Step 4. 증강(Augmentation) 실험**: `mosaic`, `scale`, `hsv_*`, `flip`
- **Step 5. 모델 크기 실험**: 모델 스케일 변경 (`nano`, `small`, `medium`)
- **Step 6. 추론 임계값 실험**: 객체 탐지 기준인 `conf` (Confidence score) 및 `iou` 실험
- **Step 7. Best 조합 통합**: 좋았던 성능 지표를 보인 파라미터들만 모아서 최종 모델 및 설정 도출

### 6.4 실험 기록 방식

모든 테스트 진행 시에는 로그에 다음 항목들을 필수로 기록해야 합니다:

## 9. YOLOv8L vs YOLOv8-World 모델 비교 실험

광고 서비스에 최적화된 객체 탐지 모델 선정을 위해,
기존 **YOLOv8L**(고정 COCO 클래스)과 **YOLOv8-World**(오픈 보캐뷸러리)를
동일한 **MVP 15 클래스** 기준으로 비교한다.

### 두 모델 핵심 차이점

| 항목               | YOLOv8L                | YOLOv8-World                      |
| ------------------ | ---------------------- | --------------------------------- |
| 모델 타입          | 고정 COCO 80 클래스    | 오픈 보캐뷸러리 (텍스트 프롬프트) |
| 클래스 지정 방식   | class ID 필터 (숫자)   | 텍스트 입력 (자유 문자열)         |
| 커스텀 클래스 추가 | ❌ 불가 (COCO 80 한정) | ✅ 가능 (자유 텍스트로 확장)      |
| 학습 방식          | 지도학습 (COCO 라벨)   | CLIP 기반 비전-언어 정렬          |
| 사용 모델 파일     | yolov8l.pt             | yolov8l-worldv2.pt                |

### 실험 설정

| 항목          | 설정                                                                                                                                  |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| 검증 데이터셋 | COCO128 (Ground Truth 기반)                                                                                                           |
| 샘플 영상     | SampleVideo_Scenes (218 scene clips)                                                                                                  |
| 공통 파라미터 | conf=0.25, iou=0.7, imgsz=640                                                                                                         |
| 평가 클래스   | MVP 15 클래스 (dog, handbag, bottle, cup, bowl, apple, chair, couch, bed, dining table, tv, laptop, remote, cell phone, refrigerator) |

### COCO128 Validation 결과 (MVP 15 클래스 기준)

| 모델             | Precision | Recall | F1 Score |
| ---------------- | --------- | ------ | -------- |
| **YOLOv8L**      | 0.7837    | 0.7886 | 0.7861   |
| **YOLOv8-World** | 0.6473    | 0.5970 | 0.6212   |

### SampleVideo 218씬 추론 성능 결과

| 모델             | 총 탐지 수 | 소요 시간(s) | 탐지/sec |
| ---------------- | ---------- | ------------ | -------- |
| **YOLOv8L**      | 248        | 88.6         | 2.5      |
| **YOLOv8-World** | 226        | 109.1        | 2.0      |

### 결과 해석

- **정확도 우위 → YOLOv8L**: MVP 15 클래스 기준 F1 Score 0.7861로, YOLOv8-World(0.6212) 대비 약 26% 높은 정확도를 기록하였다.
- **속도 우위 → YOLOv8L**: SampleVideo 218씬 기준 2.5 fps로, YOLOv8-World(2.0 fps) 대비 약 25% 빠른 처리 속도를 보였다.
- **YOLOv8-World 강점**: 고정 클래스에 국한되지 않아 향후 광고 도메인 특화 클래스(예: "라면 봉지", "맥주캔")를 텍스트 한 줄로 추가 가능.
- **최종 결론**: 정확도 및 처리 속도 모두에서 **YOLOv8L이 우위**를 보임에 따라, 다음 단계(inference 파라미터 튜닝)의 베이스 모델로 **YOLOv8L을 확정**한다.

## 10. Inference 파라미터 튜닝 - conf (Confidence Threshold)

YOLOv8L + MVP 15 클래스 기준으로 conf(Confidence Threshold) 값을 변경하며 정확도와 탐지량 변화를 측정하였다.

### 실험 설정

| 항목          | 설정                                 |
| ------------- | ------------------------------------ |
| 모델          | yolov8l.pt                           |
| 클래스 범위   | MVP 15                               |
| 검증 데이터셋 | coco128.yaml (Ground Truth 기반)     |
| 샘플 영상     | SampleVideo_Scenes (218 scene clips) |

## 15. 침묵 탐지 (Silence Detection) — 광고 삽입 타이밍 최적화

### 목적

VOD 광고 삽입 타이밍을 자동으로 결정하기 위해, 영상의 오디오에서 **침묵 구간**을 정확히 탐지한다.
기존 모션 타임라인(OpenCV)과 침묵 탐지 결과를 결합하여 **움직임 적고 소리 없는 구간**을 최적 광고 타이밍으로 확정한다.

```
최적 광고 타이밍 = 모션 낮은 구간 (기존) ∩ 침묵 구간 (신규)
```

### 모델 후보 비교

| 모델            | 방식              | 정확도      | 속도 | 특징                                        |
| --------------- | ----------------- | ----------- | ---- | ------------------------------------------- |
| **librosa**     | RMS 에너지 기반   | ⭐⭐⭐      | 빠름 | 구현 간단, 음악·효과음도 소리로 인식        |
| **pydub**       | dBFS 기반         | ⭐⭐⭐      | 빠름 | librosa와 유사한 수준                       |
| **webrtcvad**   | Google VAD        | ⭐⭐⭐⭐   | 빠름 | 음성만 탐지, 음악은 침묵으로 처리           |
| **Silero VAD**  | 딥러닝 (PyTorch)  | ⭐⭐⭐⭐⭐ | 보통 | 현재 SOTA, 음성/비음성 정밀 구분, GPU 가속  |

### 모델 선택 근거

드라마·요리 영상은 배경음악이 깔려도 **대사(음성)가 없는 순간**이 광고 삽입에 적합하다.
단순 에너지 기반(librosa/pydub)은 배경음악도 소리로 인식하여 광고 타이밍을 놓칠 수 있으므로,
**음성 유무를 정밀하게 구분하는 Silero VAD를 메인 모델로 선정**하고, librosa를 Baseline으로 삼아 비교한다.

### 실험 설정

| 항목          | 설정                                    |
| ------------- | --------------------------------------- |
| 테스트 영상   | SampleVideo.mp4 (9분 44초)              |
| Baseline 모델 | librosa (RMS, top_db 튜닝)              |
| 비교 모델     | Silero VAD (딥러닝 기반)                |
| 실행 환경     | Google Colab (T4 GPU)                   |
| 평가 기준     | 침묵 구간 수, 평균 침묵 길이, 광고 후보 타이밍 수 |

### 튜닝 파라미터

| 모델        | 파라미터         | 범위        | 설명                            |
| ----------- | ---------------- | ----------- | ------------------------------- |
| librosa     | `top_db`         | 20 ~ 60 dB  | 낮을수록 작은 소리도 소리로 인식 |
| librosa     | `min_silence_s`  | 0.5 ~ 3.0초 | 최소 침묵 길이 기준             |
| Silero VAD  | `threshold`      | 0.3 ~ 0.7   | 음성 판단 확률 임계값           |
| Silero VAD  | `min_silence_ms` | 300 ~ 1000ms| 최소 침묵 길이 기준             |

### 실험 단계별 계획

| 순서 | 대상 | 핵심 파라미터 | 기준 영상 |
| ---- | ---- | ------------ | --------- |
| 1 | OpenCV (모션 탐지) | threshold, frame_interval | SampleVideo.mp4 |
| 2 | librosa (침묵 탐지) | top_db, min_silence_s | SampleVideo.mp4 |
| 3 | Silero VAD (음성 탐지) | threshold, min_silence_ms | SampleVideo.mp4 |
| 4 | librosa vs Silero VAD | 최적값 기준 성능 비교 → 모델 1개 선정 | SampleVideo.mp4 |
| **5** | **OpenCV + 선정 모델 검증** | **확정된 최적값 고정** | **자취방 + 요리** |
| 6 | 최종 결합 | OpenCV 최적값 + 침묵 탐지 최적값 | 3개 영상 전체 |

> 1~4단계: SampleVideo.mp4 기준 파라미터 튜닝
> 5단계: 다른 영상 2종으로 일반화 검증 (YOLO 튜닝과 동일한 구조)
> 6단계: 3개 영상 전체에 최종 적용 → 광고 타이밍 자동 결정

### 튜닝 파라미터 범위

| 모델       | 파라미터             | 튜닝 범위          | 설명                              |
| ---------- | -------------------- | ------------------ | --------------------------------- |
| OpenCV     | `threshold`          | 10 ~ 60            | 낮을수록 작은 움직임도 탐지       |
| OpenCV     | `frame_interval`     | 1 ~ 30 프레임      | 몇 프레임마다 비교할지            |
| librosa    | `top_db`             | 20 ~ 60 dB         | 낮을수록 작은 소리도 소리로 인식  |
| librosa    | `min_silence_s`      | 0.5 ~ 3.0초        | 최소 침묵 길이 기준               |
| Silero VAD | `threshold`          | 0.3 ~ 0.7          | 음성 판단 확률 임계값             |
| Silero VAD | `min_silence_duration_ms` | 300 ~ 2000ms  | 최소 침묵 길이 기준               |

### 단계별 결과

#### Step 1. OpenCV 모션 탐지 튜닝

| threshold | frame_interval | 총 프레임 수 | 저모션 구간 수 | 평균 모션 점수 | 저모션 평균 점수 |
| --------: | -------------: | -----------: | -------------: | ------------: | --------------: |
|        10 |              1 |       35,026 |         21,175 |      36,201.0 |             0.0 |
|        10 |              5 |        7,005 |          2,102 |     165,851.6 |        14,539.4 |
|        25 |              1 |       35,026 |         22,299 |      17,908.1 |             0.0 |
|        25 |              5 |        7,005 |          2,102 |      88,284.6 |         1,910.8 |
|        30 |              5 |        7,005 |          2,102 |      76,212.6 |         1,079.4 |
|        40 |              5 |        7,005 |          2,102 |      58,693.8 |           350.9 |
|        50 |              5 |        7,005 |          2,103 |      46,467.1 |           107.5 |

**결과 해석:**
- `frame_interval=1`은 연속 프레임 간 차이가 너무 작아 저모션 평균 점수가 0으로 수렴 → 의미 없음
- `frame_interval=5`가 적절한 시간 해상도를 제공하며, `threshold=25~40` 범위에서 저모션 구간을 안정적으로 탐지
- `threshold`가 높을수록 저모션 평균 점수가 낮아져 더 정적인 구간만 선별됨

#### Step 2. librosa 침묵 탐지 튜닝

| top_db | min_silence_s | 침묵 구간 수 | 총 침묵 시간(s) | 평균 침묵 길이(s) | 침묵 비율(%) |
| -----: | ------------: | -----------: | --------------: | ----------------: | -----------: |
|     20 |           0.5 |           51 |            55.0 |              1.08 |       **9.4** |
|     20 |           1.0 |           22 |            34.5 |              1.57 |          5.9 |
|     25 |           0.5 |           33 |            33.2 |              1.01 |          5.7 |
|     25 |           1.0 |           12 |            17.5 |              1.46 |          3.0 |
|     30 |           0.5 |           14 |            13.6 |              0.97 |          2.3 |
|     35 |           0.5 |            6 |             5.6 |              0.93 |          1.0 |
|     40 |           0.5 |            2 |             1.9 |              0.94 |          0.3 |
|     50+ |          any |          0~1 |           0~1.3 |                 - |       0~0.2 |

**결과 해석:**
- **최대 침묵 비율이 9.4%** (top_db=20, min_silence_s=0.5)로 매우 낮음
- 드라마 영상에 BGM이 지속적으로 깔려 있어 **"소리 자체가 없는 구간"** 이 거의 존재하지 않음
- top_db=30 이상에서는 침묵 구간이 급격히 감소 → 광고 타이밍 후보로 사용 불가
- **librosa는 RMS 에너지 기반이므로 BGM이 있는 VOD 콘텐츠에 부적합**

#### Step 3. Silero VAD 튜닝

| threshold | min_silence_ms | 침묵 구간 수 | 총 침묵 시간(s) | 평균 침묵 길이(s) | 침묵 비율(%) |
| --------: | -------------: | -----------: | --------------: | ----------------: | -----------: |
|       0.3 |            500 |           91 |           222.3 |              2.44 |         38.0 |
|       0.3 |          1,000 |           55 |           191.9 |              3.49 |         32.8 |
|       0.3 |          1,500 |           40 |           169.2 |              4.23 |         29.0 |
|       0.4 |            500 |           97 |           238.9 |              2.46 |         40.9 |
|       0.4 |          1,000 |           58 |           208.2 |              3.59 |         35.6 |
|       0.5 |            500 |           96 |           249.8 |              2.60 |         42.7 |
|       0.5 |          1,000 |           59 |           221.8 |              3.76 |         38.0 |
|       0.5 |          1,500 |           44 |           198.9 |              4.52 |         34.0 |
|       0.6 |          1,000 |           60 |           230.0 |              3.83 |         39.4 |
|       0.7 |          1,000 |           57 |           233.7 |              4.10 |         40.0 |

**결과 해석:**
- BGM이 깔려있어도 **"사람 말소리가 없는 구간"** 을 정확하게 탐지
- 침묵 비율이 **25~47%** 범위로 광고 타이밍 후보가 충분함
- `threshold=0.3~0.5`, `min_silence_ms=500~1500` 범위가 침묵 비율 30~40%로 적절
- `min_silence_ms`가 클수록 짧은 침묵을 무시하여 구간 수는 줄고 평균 길이가 증가

#### Step 4. librosa vs Silero VAD 비교 → 최종 모델 선정

| 비교 항목         | librosa (최대)      | Silero VAD (중간값) | 비고                          |
| ----------------- | ------------------- | ------------------- | ----------------------------- |
| 최대 침묵 비율    | 9.4%                | **38.0%**           | Silero 4배 이상               |
| 최대 침묵 구간 수 | 51개                | **91개**            | Silero가 약 2배               |
| BGM 대응          | 불가                | **가능**            | librosa는 BGM을 소리로 인식   |
| 탐지 기준         | 소리 에너지(RMS)    | **음성 유무**       | VOD 콘텐츠에는 음성 기준 적합 |

**최종 선정: Silero VAD**

- librosa는 BGM이 있는 VOD 콘텐츠에서 침묵 구간을 거의 찾지 못함 (최대 9.4%)
- Silero VAD는 BGM과 무관하게 음성이 없는 구간을 정확히 탐지 (30~40%)
- **결론: 침묵 탐지 모델로 Silero VAD를 확정**
- OpenCV 모션 탐지, Whisper 소리 탐지, 장면 분석 모델과 함께 멀티 모델 스코어링으로 광고 타이밍 결정 예정

#### Step 5. OpenCV + Silero VAD — 다른 영상 2종 검증

확정 파라미터 (SampleVideo 기준 튜닝 결과):
- OpenCV: threshold=30, frame_interval=5
- Silero VAD: threshold=0.5, min_silence_ms=1000

| 영상        | 영상 길이 | 저모션 구간 | 침묵 구간 | 침묵 비율 | 광고 후보 | 처리 시간 |
| ----------- | --------: | ----------: | --------: | --------: | --------: | --------: |
| SampleVideo |     584s  |       117개 |      53개 |    38.1%  |      40개 |   444.1s  |
| 자취방      |     617s  |        63개 |       4개 |  **1.9%** |    **2개** |    44.8s  |
| 요리        |     597s  |        84개 |      29개 |    19.0%  |      17개 |    38.4s  |

**결과 해석:**

- **SampleVideo (드라마)**: 장면 전환·대사 끊김이 빈번하여 침묵 비율 38.1%, 광고 후보 40개로 충분
- **요리 (안성재)**: 요리 동작 중 말하지 않는 구간이 있어 침묵 19.0%, 광고 후보 17개로 보통 수준
- **자취방 (원룸)**: 나레이션이 거의 쉬지 않고 계속되어 침묵 비율 **1.9%**, 광고 후보 **2개**로 매우 부족

**일반화 성능 분석:**

| 영상 유형 | 오디오 특성 | Silero VAD 성능 | 평가 |
| --------- | ----------- | --------------- | ---- |
| 드라마     | 대사 + BGM + 효과음, 장면 전환 多 | 침묵 38.1% | ✅ 우수 |
| 요리 유튜브 | 나레이션 + 요리 소음 + BGM | 침묵 19.0% | ⚠️ 보통 |
| 자취방 유튜브 | 나레이션 거의 연속 | 침묵 1.9% | ❌ 부족 |

- SampleVideo 기준 튜닝 파라미터가 **영상 특성(나레이션 비율)에 따라 성능 편차가 큼**
- 나레이션 위주 유튜브 영상에서는 `min_silence_ms`를 줄이거나 `threshold`를 올려 더 민감하게 탐지하는 추가 튜닝이 필요할 수 있음
- 또는 영상 유형별 프리셋(드라마용 / 유튜브용)을 분리하는 방안도 고려 가능
- 단, 최종 광고 타이밍은 침묵 탐지 단독이 아닌 **멀티 모델 스코어링**(OpenCV + Silero VAD + Whisper + 장면 분석)으로 결정하므로, 침묵 비율이 낮더라도 다른 모델의 점수로 보완 가능

## 17. 장면 감정 분석 정확도 개선 — 멀티 입력 기반 Gemini 분석

### 배경 및 문제점

기존 장면 분석은 **프레임 이미지 1장만** Google Vision에 전달하여 상황/감정/욕구를 추출하였다.
그 결과 맥락 부족으로 인해 **분석 정확도가 낮은 문제**가 있었다.

이를 개선하기 위해 **객체 정보(YOLO) + 대사 텍스트(Whisper) + 프레임 이미지**를 함께 Gemini LLM에 전달하는 멀티 입력 방식으로 전환한다.

### 기존 vs 개선 비교

| | 기존 방식 | 개선 방식 |
|--|----------|----------|
| **입력** | 프레임 이미지 1장 | 프레임 3장 + 대사 + 객체 |
| **분석 모델** | Google Vision (라벨 추출) | Gemini LLM (멀티모달) |
| **대사 활용** | 없음 | Whisper 추출 대사 포함 |
| **객체 활용** | 없음 | YOLOv8L 탐지 객체 포함 |
| **씬 분할 기준** | scenedetect (영상 화면 변화) | Whisper (대사 문맥 변화) |
| **정확도** | 낮음 (맥락 부족) | 높음 (멀티 입력) |

### 작업 분담

| 담당 | 작업 | 산출물 |
| ---- | ---- | ------ |
| 본인 | YOLOv8L 씬별 객체 탐지 | `hyuck1_scene_objects.csv` (객체 + 타임스탬프) |
| 팀원 | Whisper 대사 추출 + 문맥 기준 씬 분할 | 대사 CSV (텍스트 + 타임스탬프) + 씬 분할 영상 |

### Gemini 분석 파이프라인

```
[입력 1] YOLOv8L 객체 (타임스탬프)
  → "chair, cup, tv, cell phone"

[입력 2] Whisper 대사 (타임스탬프)
  → "오늘 진짜 힘들었다. 커피 한 잔 하자."

[입력 3] 씬 프레임 이미지 3장 (시작 + 중간 + 끝)
  → Whisper 문맥 기준 분할 구간에서 캡처

        ↓ Gemini API 호출

[출력] 상황 / 감정 / 욕구 (context_narrative)
```

### 프레임 선택 기준

| 프레임 | 위치 | 목적 |
| ------ | ---- | ---- |
| 시작 프레임 | 구간 시작 + 0.5s | 장면 도입부 배경/인물 확인 |
| 중간 프레임 | 구간 중앙 | 핵심 행동/표정 캡처 |
| 끝 프레임 | 구간 끝 - 0.5s | 장면 마무리/전환 확인 |

### 타임스탬프 매핑 방식

YOLO 객체는 scenedetect 기준(1669개 씬)으로 추출되었으므로, Whisper 문맥 기준 씬과 타임스탬프 범위를 매핑하여 병합한다.

```python
# Whisper 씬 구간 내에 포함되는 YOLO 객체를 합침
Whisper 씬 1: 0.0s ~ 15.0s
  → YOLO CSV에서 start_sec이 0.0 ~ 15.0 범위인 행의 객체를 병합
  → "chair, cup, tv, cell phone, bottle"
```

### Gemini 프롬프트

```python
_PROMPT_TEMPLATE = (
    "아래는 한국 TV 드라마의 한 씬에 대한 정보입니다.\n\n"

    "감정: 이 씬이 시청자에게 전달하는 감성적 분위기나 정서를 표현하세요. (상황 묘사 금지)\n"
    "욕구: 이 씬이 타겟하는 시청자의 내면적 니즈나 욕구를 서술하세요. (감정 단어·상황 묘사 금지)\n\n"
    "예시:\n"
    "상황: 퇴근 후 집에 돌아온 직장인이 소파에 앉아 따뜻한 음료를 마시는 장면이다.\n"

# 호출 예시
prompt = _PROMPT_TEMPLATE.format(
    context_block="씬 이미지 설명",

    dialogue_text="오늘 진짜 힘들었다. 커피 한 잔 하자."

```

## 18. 광고-씬 매칭 정확도 분석 및 개선 계획

### 현재 문제점

Step 4(Decision)에서 광고-씬 매칭 유사도가 **최대 0.37 수준**으로 매우 낮아, 씬과 맥락적으로 어울리지 않는 광고가 배치되는 문제가 발생하고 있다.

#### 매칭 구조

현재 매칭은 `context_narrative`(씬)와 `target_narrative`(광고)를 **MiniLM 임베딩 모델(`jhgan/ko-sroberta-multitask`)**로 각각 384차원 벡터로 변환한 뒤, 코사인 유사도 1개로 비교하는 방식이다.

```
context_narrative (씬 - Gemini 생성):
  상황: 사장이 되기 전에 강원주라는 이름으로 입사한 회사에서 면접을 보는 장면이다.
  감정: 긴장감과 불안함이 감도는 분위기를 전달한다.
  욕구: 안정적인 직장과 사회적 인정을 얻고 싶은 사람에게 어울린다.

target_narrative (광고 - Gemini 생성):
  상황: 퇴근 후 집에 돌아온 직장인이 소파에 앉아 따뜻한 음료를 마시는 장면이다.
  감정: 하루의 피로가 녹아드는 포근하고 안도감 있는 분위기를 전달한다.
  욕구: 바쁜 일상 속에서 잠깐의 휴식과 자신을 위한 작은 여유를 원하는 사람에게 어울린다.
```

두 텍스트 모두 **상황/감정/욕구** 3차원 구조로 잘 생성되어 있으나, 전체를 하나의 벡터로 비교하면 유사도가 **0.37**에 불과하다.

#### 원인 분석

| 원인 | 설명 |
| ---- | ---- |
| **상황 텍스트 지배** | "상황:" 부분이 전체 텍스트의 60~70%를 차지하여, 임베딩 벡터가 상황 의미에 편향됨 |
| **상황의 본질적 불일치** | 씬 상황(드라마 줄거리)과 광고 상황(제품 사용 장면)은 **근본적으로 다른 내용**을 서술 |
| **감정/욕구 매몰** | 감정·욕구가 유사하더라도 상황 불일치로 인해 전체 유사도가 낮게 나옴 |
| **Cross-Encoder 미학습** | 2차 정밀 평가 모델(Cross-Encoder)이 아직 학습되지 않아 MiniLM fallback으로만 동작 |

#### 실제 유사도 테스트 결과

DB에서 조회한 실제 씬-광고 유사도 (MiniLM 코사인 유사도):

| 씬 내용 (요약) | 최고 매칭 광고 | 유사도 |
| -------------- | ------------- | -----: |
| 회사 입사 면접 장면 | 우리금융그룹 | 0.3726 |
| 사무실에서 전화 통화 | 삼성전자 | 0.3512 |
| 두 사람이 약속을 나누는 장면 | 현대자동차 | 0.3289 |

→ 모든 매칭에서 **pre-filter 임계값(0.35~0.45)에 근접하거나 미달**하여, 적절한 광고 후보가 탈락되거나 부적절한 광고가 선택되는 상황이다.

### 개선 계획: 차원별 분리 매칭 (Dimension-wise Matching)

#### 핵심 아이디어

상황/감정/욕구를 **텍스트에서 분리**하여 각 차원을 **개별적으로 비교**하고, **가중합산**으로 최종 유사도를 산출한다.

```
기존:  전체 context_narrative ↔ 전체 target_narrative  →  유사도 1개 (0.37)

개선:  씬 상황 ↔ 광고 상황  →  sim_situation  × 0.2
       씬 감정 ↔ 광고 감정  →  sim_emotion    × 0.3
       씬 욕구 ↔ 광고 욕구  →  sim_desire     × 0.5
                                ─────────────────────
                                최종 유사도 = 가중합산
```

#### 가중치 설계 근거

| 차원 | 가중치 | 근거 |
| ---- | -----: | ---- |
| **상황** | 0.2 | 드라마 상황과 광고 상황은 본질적으로 다를 수밖에 없음. 높은 가중치를 주면 오히려 매칭 품질이 떨어짐 |
| **감정** | 0.3 | 감정적 톤의 일치가 시청 경험에 중요. "긴장감 있는 씬"에 "에너지 넘치는 광고"가 "편안한 광고"보다 어울림 |
| **욕구** | 0.5 | 광고 타겟팅의 핵심. "안정적인 삶을 원하는" 씬에 "보험/금융 광고"가 매칭되어야 함 |

#### 기대 효과

| 시나리오 | 기존 (전체 비교) | 개선 후 (차원별) | 판정 변화 |
| -------- | ---------------: | ---------------: | --------- |
| 상황 다르고 욕구 비슷 | 0.35 (탈락) | **0.55+** (통과) | ❌ → ✅ |
| 상황 비슷하고 욕구 다름 | 0.40 (통과) | 0.30 (탈락) | ✅ → ❌ (올바른 제거) |
| 전체적으로 잘 맞음 | 0.45 | **0.65+** | ✅ → ✅ (점수 상승) |

#### 수정 대상 파일

| 파일 | 변경 내용 |
| ---- | --------- |
| `embedding_scorer.py` | 텍스트를 상황/감정/욕구로 파싱하는 함수 + 차원별 비교 함수 추가 |
| `decision.py` | 기존 `batch_similarity_matrix()` 대신 차원별 매칭 함수 호출 |
| `pre_filter.py` | 차원별 매칭 시 점수 분포 변화에 맞춰 임계값 조정 |

#### 작업 상태

| 순서 | 작업 | 상태 |
| ---- | ---- | ---- |
| 1 | 현재 매칭 구조 분석 및 문제 진단 | ✅ 완료 |
| 2 | 차원별 분리 매칭 설계 | ✅ 완료 |

## 19. VLM 백엔드 비용 최적화 — Gemini → Qwen2.5-VL 대체 테스트

### 배경

Gemini 2.5 Flash API의 토큰 비용이 높아, 동일한 역할을 수행할 수 있는 무료/저비용 대체 모델을 탐색하였다.

Gemini가 담당하는 역할:
- 씬 프레임(최대 3장) + YOLO 탐지 객체 + Whisper 대사를 입력받아
- **상황/감정/욕구** 3항목 형식의 `context_narrative`를 생성

### 대체 모델 후보 조사

HuggingFace Inference API에서 무료로 사용 가능한 Image-Text-to-Text 모델을 대상으로 조사하였다.

#### 평가 기준

| 기준 | 설명 |
| ---- | ---- |
| 한국어 성능 | KMMLU 벤치마크 등 한국어 평가 지표 |
| 추론 속도 | 모델 크기, MoE 여부, 프로바이더 성능 |
| 비용 | 무료 티어 크레딧 소모량 |
| 구조화 출력 | 상황/감정/욕구 형식 프롬프트 준수 능력 |

#### 후보 모델 비교
